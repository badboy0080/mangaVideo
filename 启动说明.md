# 漫剧流水线 — 启动说明

> 面向本地开发/使用 Web 控制台。按顺序做即可。

---

## 一、你需要准备什么


| 工具               | 用途               | 怎么检查是否已安装               |
| ---------------- | ---------------- | ----------------------- |
| **Python 3.10+** | 跑流水线后端           | 终端输入 `python --version` |
| **Node.js 18+**  | 跑网页前端            | 输入 `node --version`     |
| **pnpm**         | 安装/启动前端          | 输入 `pnpm --version`     |
| **ffmpeg**       | 最后一步拼接成片（Step 8） | 输入 `ffmpeg -version`    |
| **API 密钥**       | 调用 AI 生剧本、生图、生视频 | 见下文「配置密钥」               |


**ffmpeg 安装提示（Windows）**

- 从 [ffmpeg 官网](https://ffmpeg.org/) 下载，解压后将 `bin` 目录加入系统 **PATH**
- 或在 WSL / Ubuntu：`sudo apt install ffmpeg`

## 二、第一次使用（只需做一次）

### 1. 进入项目目录

```powershell
cd D:\AIcoding\manga-pipeline\manga-pipeline
```

> 以后说的「项目根目录」都是这个文件夹。

### 2. 安装 Python 依赖

```powershell
pip install -r requirements.txt
```

### 3. 配置 API 密钥

复制环境变量模板：

```powershell
Copy-Item .env.example .env
```

用记事本或编辑器打开根目录下的 `**.env**`，至少填写：

```env
DEEPSEEK_API_KEY=你的DeepSeek密钥
ARK_API_KEY=你的火山引擎Ark密钥
```

说明：

- **DeepSeek**：Step 1～4、6 的剧本、分镜、提示词（文字类 AI）
- **ARK / VOLCENGINE**：Step 5 生图、Step 7 生视频（二选一填即可，可填同一个 Key）

### 4. 安装前端依赖

```powershell
cd web
pnpm install
cd ..
```

## 三、日常启动（每次要用时）

需要 **两个终端窗口**，都保持在 **项目根目录**。

### 终端 1：启动后端 API

```powershell
cd D:\AIcoding\manga-pipeline\manga-pipeline
python -m uvicorn server.main:app --host 127.0.0.1 --port 8765
```

看到类似 `Uvicorn running on http://127.0.0.1:8765` 即表示成功。**不要关闭这个窗口。**

**自检：** 浏览器打开 [http://127.0.0.1:8765/api/health](http://127.0.0.1:8765/api/health)  
应返回 JSON，且 `"ok": true`。

### 终端 2：启动 Web 前端

```powershell
cd D:\AIcoding\manga-pipeline\manga-pipeline\web
pnpm dev
```

终端会显示本地地址，一般是：

- [http://localhost:5173](http://localhost:5173)

在浏览器打开该地址即可使用控制台。

## 四、在网页里怎么用

1. **首页**：输入创意主题，选风格和时长，创建项目
2. **我的项目 / 最近项目**：点卡片进入某个项目
3. **流水线页**：
  - 顶部 **8 个圆点**：切换步骤（剧本纲要 → 分镜 → … → 成片）
  - **产物预览**：查看或编辑当前步骤文档
  - **运行此步骤 / 运行全部步骤**：让 AI 继续跑流水线
  - **运行日志**：展开可看后台输出

产物文件保存在：`outputs/<项目ID>/` 目录下。

## 五、常见问题

### 1. 网页提示「无法连接 API」

- 确认 **终端 1 的后端还在运行**
- 确认后端是在 **项目根目录** 启动的（不是别的副本文件夹）
- 浏览器访问 [http://127.0.0.1:8765/api/health](http://127.0.0.1:8765/api/health) 是否正常

### 2. 改了代码但网页没变化

- 前端：在运行 `pnpm dev` 的终端按 `Ctrl+C` 停止，再重新 `pnpm dev`
- 或浏览器 **硬刷新**：`Ctrl + F5`
- 后端改了 `server/` 代码：需重启终端 1 的 uvicorn

### 3. 流水线某步一直失败

- 检查 `.env` 里 Key 是否填对、是否有余额
- 展开该步 **运行日志** 看具体报错
- 生图/生视频步骤需要 **ARK_API_KEY**；文字步骤需要 **DEEPSEEK_API_KEY**

### 4. 最后一步没有成片 `final.mp4`

- 确认本机已安装 **ffmpeg** 且命令行能执行 `ffmpeg -version`
- 确认 Step 7 已成功生成 `outputs/<项目ID>/videos/` 下的视频

### 5. 页面中文变成问号 `????`

- 不要用会破坏 UTF-8 的方式编辑 `web/src/App.tsx`
- 可运行：`python scripts/restore_app_chinese.py`（见项目规则文档）

## 六、不用网页、只用命令行（可选）

在项目根目录：

```powershell
python pipeline.py "你的创意主题" --duration 90 --style 电影短片
```

会从 Step 1 跑到 Step 8，产物同样在 `outputs/<run_id>/`。

## 七、端口与地址一览


| 服务        | 地址                                 | 说明                          |
| --------- | ---------------------------------- | --------------------------- |
| 后端 API    | `http://127.0.0.1:8765`            | FastAPI                     |
| 健康检查      | `http://127.0.0.1:8765/api/health` | 确认后端版本与路由                   |
| 前端页面      | `http://localhost:5173`（常见）        | Vite 开发服务器，端口占用时会自动换 5174 等 |
| 前端 API 配置 | `web/.env.development`             | 默认已指向 `8765`，一般不用改          |
