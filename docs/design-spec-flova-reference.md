# 参考界面设计规范说明（Flova.ai 风格拆解）

本文档基于你提供的 **Flova.ai** 产品截图，整理可落地的**视觉与组件规范**，便于在本项目（漫剧流水线 Web）或其它界面中对照实现。  
**说明**：以下为从界面归纳的设计语言，**非** Flova 官方设计稿；色号与尺寸为目测估算，落地时请以浏览器取色为准。

---

## 1. 设计气质（一句话）

**深色极简 + 大块面分层 + 几乎无线框**：用「背景色差」区分区域，而不是描边；强调留白与层级清晰的字号对比。

---

## 2. 布局结构

| 区域 | 特征 |
|------|------|
| **背景** | 近黑底色，顶部可有极弱的彩色渐变（偏橙 / 偏蓝的氛围光），不占主体。 |
| **左侧栏** | 窄竖条导航：图标 + 文字标签；当前项用「圆形底」或浅色块高亮，而非粗边框。 |
| **顶栏** | 左侧品牌 Logo + 文案；右侧次要操作（教程、注册类按钮）。 |
| **主内容区** | **居中**、宽度受限（大留白）；核心是一块「超大输入区」+ 下方横向内容带（如热门卡片）。 |

**与本项目映射**：你可保留「左侧生成历史 + 右侧主区」的双栏结构，把主区内参考这里的**居中卡片式输入区**与**下方横向卡片栅格**。

---

## 3. 色彩规范（估算）

| 角色 | 用途 | 参考值（深色模式） |
|------|------|-------------------|
| **页面底** | 整页背景 | `#000000` ~ `#0A0A0A` |
| **组件面** | 大输入框、卡片 | `#161616` 左右，略亮于页面底即可 |
| **主文案** | 标题、正文 | `#FFFFFF` 或极高对比浅灰 |
| **次级文案** | 说明、辅助 | `#A0A0A0` 一类（约 60%～70% 视觉权重） |
| **分割线** | 若有弱分隔 | **浅灰细线**，如 `#333333`～`#444444`，或低透明度白色（与本项目已有 `border-line-soft` 思路一致） |
| **强调色** | CTA、教程按钮 | 青绿渐变或小面积纯色按钮，避免整屏艳色 |

**原则**：尽量少用「描边区分块区」；优先 **`组件背景 > 页面背景`** 形成层级。

---

## 4. 字体与排版

- **字体**：无衬线、现代感（界面类似 Inter / SF / 系统等宽比例友好的几何无衬线）。
- **层级**：
  - 主标题：大号、偏粗、白色。
  - 区块标题（如「热门 Skills」）：小于主标题，仍清晰。
  - 描述：小一号、灰色，行距略松。
- **对齐**：主创意区标题与输入宜**视觉居中**，增强「工作台焦点」感。

---

## 5. 核心组件规范

### 5.1 主输入区（创意输入）

| 属性 | 建议 |
|------|------|
| **形状** | 大圆角矩形（约 **20px～30px** 圆角，按屏幕可调）。 |
| **背景** | 比页面略浅的深色面（见上表「组件面」）。 |
| **边框** | **无明显线框**；可选极弱内阴影或仅靠底色对比。 |
| **占位文案** | 语气引导型（参考：「由一个想法成故事开始…」）。 |
| **底部工具行** | 左对齐：**胶囊形（pill）** 小按钮（如 Model / Skill / Element）+ 「+」；右侧可放**圆形主行动按钮**（如向上箭头发送）。 |

**与本项目**：你的「风格类型 / 时长」若做成下拉，可放在输入区**底部一行**，样式对齐 **pill** 或与 Flova 底部控件同一视觉密度。

### 5.2 胶囊按钮（Pill）

- 小圆角或全圆角药丸形。
- 背景略亮于输入区内底，或与输入区内一条「工具栏」同色。
- **避免**明显描边；可用 hover 时轻微变亮区分。

### 5.3 横向卡片（如「热门 Skills」）

| 属性 | 建议 |
|------|------|
| **排列** | 横向滑动或网格，多张并列。 |
| **卡片** | 圆角矩形、`#161616` 类表面，**无线框**。 |
| **内容** | 小图标 + **粗标题** + **一行灰色简介**。 |
| **间距** | 卡片之间留白充足，避免挤在一起。 |

**与本项目**：「我的项目」成片缩略图栅格可对齐：**圆角 + 无底框 + 仅靠底色与 hover 提亮**。

### 5.4 顶栏按钮

- **主要 CTA**：可有轻微渐变（蓝绿），圆角矩形。
- **次要按钮**：深灰填充 + **若有边线则极细极淡**（你当前产品偏好可改为**无边线**，仅用底色差）。

### 5.5 侧栏导航

- 图标线性或较细填充，默认浅灰。
- **选中态**：圆形底或浅色块衬托，而非粗边框。

---

## 6. 与本项目既定规则的对齐

你已强调的方向可与 Flova 归纳一致：

1. **细线**：分割线用**浅灰、低对比**（参见色彩表中分割线一行；实现上可用 `color-mix` / `border-line-soft` 一类语义）。
2. **取消线框**：卡片、按钮、输入区优先 **borderless**，用 **`surface` 色差**表达层级。
3. **下拉**：风格 / 时长用下拉时，样式与 pill / 工具条统一高度与圆角，避免「系统默认灰框」突兀。

---

## 7. 落地检查清单（给设计与前端）

- [ ] 页面背景与「输入区 / 卡片」是否有 **清晰但克制** 的明度差？
- [ ] 是否 **极少出现 1px 强对比描边**？
- [ ] 主文案 / 次级文案是否至少有 **两级字号 + 两级颜色**？
- [ ] 圆角是否在 **输入区、卡片、按钮** 之间大致同一「家族」（不要混用直角与大圆角无规律）？
- [ ] CTA 是否 **小面积出现**，避免满屏渐变？
