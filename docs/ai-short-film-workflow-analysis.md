# AI 短片创作流程分析报告

日期：2026-05-28  
分析对象：`视频创作流程2.docx`  
对照项目：当前 `manga-pipeline` 短片流水线

## 一、结论先说

这份文档里的 AI 创作方式，本质上是一个“导演助理式流程”：用户先给完整创意、分镜、风格和模型要求，AI 再拆成创意概要、视觉策略、关键元素、静态图、视频、音乐、时间线组装等阶段。

它和当前项目最大的差别是：文档里的流程更像“创作工作台”，强调边做边确认、风格锚点、模型切换、分批生成和人工调度；当前项目更像“自动流水线”，强调从主题一键跑完剧本、分镜、资产、视频和拼接。

如果要让当前项目更接近文档里的体验，重点不是单纯多接几个模型，而是补三类能力：

1. 全局视觉规格：把“暗黑写实”“Kodak Vision3”“胶片颗粒”“人物一致性”等作为可继承的项目级设定。
2. 分批确认机制：允许用户在关键元素、第一批镜头、视频批次、音频、组装前逐段确认或跳过确认。
3. 多模型路由：不同项目可选择 Midjourney、GPT Image、Seedream、Kling、Hailuo、HappyHorse、Suno 等，而不是固定一套图生图/图生视频后端。

## 二、文档里的对话结构

文档包含两个主要案例。

### 案例 1：《悟空·暗渊》暗黑预告片

用户先给出高度完整的创作输入：

- 片名与主题：神性与魔性的挣扎、英雄孤独与宿命。
- 总时长：40 秒。
- 视觉风格：电影级 CG、暗黑奇幻、光影对比、材质细节。
- 音乐风格：空灵吟唱、鼓点、史诗交响、钟声/心跳收尾。
- 分镜序列：0-8 秒、8-18 秒、18-30 秒、30-40 秒。
- 每段都有画面描述、音乐音效、Midjourney 提示词。
- 还有核心提示词词汇库。

AI 的返回格式很稳定，基本分成：

- `Creative Brief已完成`：项目创意概要。
- `故事板 已更新`：分镜结构已进入系统。
- `Visual Gen Path已完成`：视觉生成策略。
- `Visual Assets已完成`：图像或视频资产生成完成。
- `Music & Sound已完成`：音乐生成完成。
- `时间线已完成`：视频组装完成。

这说明它不是只返回文本，而是在模拟一个有状态的创作系统：每一步都写入一个内部模块，例如规格、故事板、视觉资产、音乐、时间线。

### 案例 2：《潮汕·时间的气味》人文纪录片

用户给的是更偏“项目需求文档”的输入：

- 模型：happyhorse。
- 时长：约 3 分钟。
- 类型：人文纪录片风格 TVC。
- 主题：潮汕本土生活与文化。
- 结构：开场、第一章、第二章、第三章、结尾。
- 分镜：按段落写画面、镜头关键词、旁白。
- 静态素材：GPT image 2。
- 视频生成：HappyHorse。
- 声音：环境音、极简钢琴、轻民乐。

AI 的流程变成：

- 加载“人文纪实电影工作流”。
- 建立视频规格。
- 设计 15 个镜头。
- 定义 5 个关键视觉元素。
- 先生成元素参考图。
- 分批生成视频：先开场 + 第一章，再剩余 9 个镜头。
- 根据用户反馈更新 Kodak Vision3 色彩锚点。
- 生成 BGM 和旁白。
- 粗剪合成。
- 继续做海报。

这个案例体现了一个很重要的产品能力：用户中途补充的风格要求，会升级成“全局锚点”，影响后续所有镜头。

## 三、AI 返回文本格式分析

文档里的 AI 返回文本不是普通聊天，而是“状态反馈 + 下一步动作 + 等待确认”的格式。

### 1. 状态块

常见格式：

- `Creative Brief已完成`
- `Visual Gen Path已完成`
- `Visual Assets已完成`
- `Media Assets已完成`
- `Music & Sound已完成`
- `时间线 已更新`

优点：用户很容易知道当前进行到哪一步。

缺点：这些状态不够结构化。如果要做成产品，最好在后端保存成 JSON，例如：

```json
{
  "stage": "visual_assets",
  "status": "completed",
  "asset_ids": ["Shot_WukongHandClench_img"],
  "next_action": "generate_next_static_image"
}
```

### 2. 策略说明块

AI 会在关键节点解释自己的计划，例如：

- 先生成核心角色参考图。
- 再为每个镜头生成第一帧静态图。
- 然后用视频模型转成动态视频。
- 因模型时长固定，最终需要剪辑裁剪。
- 使用角色参考图保证一致性。

这类文字对产品经理很友好，但对系统执行不够友好。建议拆成“给用户看的解释”和“给系统执行的参数”两层。

### 3. 确认机制

文档里的默认节奏是：

- 生成关键元素后暂停确认。
- 生成单张静态图后暂停确认。
- 用户可以说“后面全部生成”，AI 才切换成批量模式。

这比当前项目的一键自动流水线更适合创作，因为视频创作里“方向错了继续跑”会浪费很多时间和额度。

## 四、文生图提示词分析

文档里的文生图提示词有两种形态。

### 1. Midjourney 英文提示词

案例 1 用的是英文 Midjourney 风格：

- 主体：Sun Wukong / Water Curtain Cave。
- 风格：dark fantasy / epic / cinematic。
- 质感：hyperdetailed / hyper-realistic / 8k。
- 光影：cinematic lighting / volumetric light / rim light。
- 渲染词：unreal engine 5 / octane render / V-Ray。
- 参数：`--ar 16:9 --style raw`。

问题在于，用户后来明确说“不要渲染图，要暗黑风格、电影质感、史诗级照片、超细腻写实”。这说明最初的提示词里 `unreal engine 5`、`octane render`、`V-Ray` 这类词，会把画面带向 CG 渲染感，不适合用户后来的真实摄影方向。

更好的格式应该是：

```text
暗黑史诗电影摄影风格，超写实照片质感，低调光影，强明暗对比，深黑与暗金色调，体积光穿过潮湿洞穴空气，画面主体是暗黑孙悟空，破损暗金护腕与战损铠甲，毛发细节清晰，眼神疲惫而压抑，16:9，no text, no watermark
```

### 2. GPT Image 2 / 纪实风统一模板

案例 2 的提示词更像“风格模板”：

```text
photo realistic documentary style, Chaoshan China local life, natural light, soft warm tone, film grain, cinematic composition, slow life, humanistic photography, award winning documentary style, 35mm film look
```

它的优点是统一、克制、容易复用；缺点是每个镜头还需要补充主体动作、构图、环境细节，否则容易变成泛泛的纪录片照片。

推荐结构：

```text
全局风格 + 地点 + 主体 + 动作/状态 + 镜头景别 + 光线 + 色彩 + 质感 + 禁止项
```

## 五、图生视频提示词分析

文档里的图生视频提示词重点不是重新描述画面，而是描述“这张图怎么动起来”。

好的图生视频提示词包含：

- 镜头运动：缓慢推近、固定镜头、横移、拉远、下摇。
- 主体动作：手缓缓握拳、转头直视镜头、能量升腾、倒茶、走远。
- 环境动态：水滴、雾气、尘埃、灯亮起、风吹披风。
- 时长匹配：6 秒、10 秒、15 秒固定时长。
- 音频提示：心跳、风声、瀑布、钢琴、旁白。

技术上要特别注意：图生视频提示词不要重复太多静态画面细节，因为参考图里已经有了。它应该更关注“动态增量”。当前项目的 `prompts/step_06_i2v_polish_system.txt` 已经有类似规则：删除首帧已有静态描述、保留 `@资产名`、按时长控制动作密度。这一点和文档里的优秀实践是对齐的。

## 六、AI 短片创作完整流程

从文档归纳，完整流程应该是：

1. 输入创意需求：主题、时长、风格、模型、比例、音乐方向。
2. 生成 Creative Brief：视频类型、情绪基调、视觉风格、节奏、声音设计、受众。
3. 生成故事板/分镜：每个镜头的画面、时长、动作、声音、旁白。
4. 生成视觉策略：选择图像模型、视频模型、关键元素、参考图方式、一致性策略。
5. 生成关键元素参考图：角色、场景、道具、色彩锚点。
6. 生成每个镜头静态图：通常一镜 1-3 张备用。
7. 人工确认或批量通过：不满意则改风格锚点或重生。
8. 图生视频：按静态图生成视频片段。
9. 音乐与声音：BGM、环境音、旁白。
10. 时间线组装：视频片段 + 音频 + 旁白。
11. 导出成片。
12. 衍生物：海报、封面、宣传图。

## 七、与当前项目的差别

### 1. 当前项目已有的能力

当前项目已经有比较完整的 5 步流水线：

1. Step01 剧本：生成 `script_brief.json`。
2. Step02 分镜：生成 `storyboard.json` 和 `script.md`。
3. Step03 资产：生成角色、场景、道具图片和 `img_results.json`。
4. Step04 视频：生成 `video_prompts.json`，调用视频模型生成片段。
5. Step05 成片：拼接成 `final.mp4`。

项目还具备：

- Agent 编排器。
- 创意总监 Agent。
- Step1/2/3/4 审核结果。
- 图片 prompt 展示与重新生成。
- `@资产名` 参考绑定。
- I2V prompt 润色。
- 图片、视频并发。

这些基础是好的，已经超过简单脚本工具。

### 2. 当前项目弱于文档流程的地方

| 维度 | 文档里的流程 | 当前项目 |
|---|---|---|
| 创作方式 | 分阶段确认，可随时改方向 | 更偏一键流水线 |
| 模型选择 | 用户可指定 Midjourney / Kling / Hailuo / HappyHorse / Suno | 目前主要固定在现有接入模型 |
| 风格锚点 | 用户反馈会升级为全局规格 | 创意总监指导已保存，但下游未充分继承 |
| 关键元素 | 先做角色/场景/道具参考图，再做镜头 | 已有资产图，但“项目级视觉锚点”还不够突出 |
| 分批生成 | 可先生成一章或几个镜头确认 | 当前更偏按步骤整体执行 |
| 音频 | BGM、旁白、环境音作为流程一部分 | 当前主要视频拼接，音频能力较弱 |
| 海报/衍生图 | 成片后可继续做海报 | 当前没有作为正式流程 |
| 时间线编辑 | 有 Timeline 概念 | 当前以拼接结果为主，时间线可视化和音轨较弱 |

### 3. 当前项目强于文档流程的地方

当前项目更工程化：

- 有真实文件产物。
- 有后端 API。
- 有数据库记录。
- 有前端项目管理。
- 有测试。
- 有审核 Agent。
- 有失败降级和停止机制。

文档里的 AI 更像一个成熟产品的交互样例，但不一定暴露底层数据结构；当前项目虽然体验还粗一些，但更接近可维护的软件系统。

## 八、技术建议

### 优先级 1：增加“项目级规格 JSON”

建议新增一个 `creative_spec.json` 或扩展现有 `director_guidance.json`，保存：

```json
{
  "visual_style": "暗黑电影摄影写实",
  "color_anchor": "深黑、暗金、低饱和",
  "camera_language": "slow push, static camera, close-up",
  "image_model": "seedream",
  "video_model": "seedance",
  "audio_style": "空灵吟唱到史诗交响",
  "negative_prompt": "no text, no watermark, no subtitles"
}
```

然后 Step01、Step02、Step03、Step04 都读取它，而不是只保存给前端展示。

### 优先级 2：做“分批确认模式”

建议增加三种运行模式：

- 自动跑完：适合测试和低成本 demo。
- 关键节点确认：关键元素、首批镜头、视频、音频、组装前确认。
- 全人工确认：每张图/每个视频都确认。

这样更符合真实创作流程。

### 优先级 3：把图片 prompt 分成两层

当前项目已经能展示图片 prompt，但建议拆成：

- 全局风格 prompt：整片统一。
- 单镜头/单资产 prompt：当前镜头特有。

这样用户说“全部图片不要渲染图”时，只改全局风格，不需要每张图都改。

### 优先级 4：增强音频与时间线

建议后续加入：

- BGM prompt。
- 旁白脚本。
- 环境音清单。
- 每个镜头的音频 cue。
- 时间线 JSON：视频片段、音频片段、开始时间、结束时间、裁剪区间。

### 优先级 5：模型路由抽象

不要把模型写死在流程里。建议做模型能力表：

```json
{
  "image": ["seedream", "gpt_image", "midjourney_manual"],
  "video": ["seedance", "hailuo", "kling_manual", "happyhorse_manual"],
  "audio": ["suno_manual", "tts", "local_placeholder"]
}
```

当前如果暂时没有 API，也可以先做“手动模型模式”：生成提示词和导入槽位，让用户去外部模型生成后上传回来。

## 九、产品层面的改进路线

建议按 4 个阶段推进：

### 阶段 1：对齐文档里的创作体验

- 增加项目级视觉规格。
- 让创意总监指导真正注入下游 prompt。
- 增加“全部后续自动生成/每步确认”的开关。

### 阶段 2：加强人工可控性

- 关键元素图支持确认、替换、锁定。
- 分镜支持按段落批量生成。
- 视频支持按镜头重生和保留历史版本。

### 阶段 3：补齐声音和时间线

- BGM prompt。
- 旁白脚本。
- 音效建议。
- 时间线 JSON 与前端预览。

### 阶段 4：多模型开放

- 图像模型可选。
- 视频模型可选。
- 音频模型可选。
- 外部生成结果可上传回填。

## 十一、最终判断

文档里的流程代表了更成熟的“AI 影视创作产品体验”：它不是只追求自动生成，而是强调用户确认、视觉一致性、分段推进、多模型协作和最终时间线。

当前项目已经具备工程底座，尤其是 5 步流水线、Agent、审核、资产绑定和视频 prompt 润色。下一步最值得做的是：把“创意指导”从展示信息变成真正控制下游生成的全局规格，并把一键流水线升级为“可确认、可分批、可回退”的创作工作台。
